✓ Solution 3

Interference: Bus ID

To ensure accurate and organized tracking, a unique identifier (Bus ID) is added to each bus. This identifier is sent with every LoRa message containing location and speed data.

The station can immediately identify the source of the message and classify it according to the correct bus.

✓ Solution 5

Map Distance: Haversine

Mathematical Formula

a = sin²(Δφ/2) + cos(φ₁)·cos(φ₂)·sin²(Δλ/2)
─────────────────────────
c = 2·atan2(√a, √(1−a))
─────────────────────────
d = R · c

This formula is based on the Earth's radius and provides very accurate results compared to simpler methods. It is more than sufficient for current tracking applications and is widely used in GPS and IoT systems.

haversine.cpp
// Haversine Formula
#define R 6371000 // Earth radius

double haversine(
    double lat1, double lon1,
    double lat2, double lon2
) {
    double dLat = toRad(lat2 - lat1);
    double dLon = toRad(lon2 - lon1);

    double a = sin(dLat/2)*sin(dLat/2) +
        cos(toRad(lat1))*cos(toRad(lat2))*
        sin(dLon/2)*sin(dLon/2);

    double c = 2*atan2(
        sqrt(a), sqrt(1-a));

    return R * c; // meters
}

⚡ Working Mechanism

Data Flow

1

Data Collection (The Bus)

The ESP32 unit receives the coordinates from the GPS and combines them with the bus ID.

2

LoRa Transmission

The data is transmitted via the LoRa module at a low frequency and low power over long distances.

3

Bus Stop Reception

The bus stop receives the packet and decodes it immediately.

4

Data Processing

The station processes the data directly without storing it and prepares it for display.

5

Final Display

The web interface retrieves the data and shows the bus's location, speed, and estimated arrival time.
